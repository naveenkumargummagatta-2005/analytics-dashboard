> A full-stack data pipeline that reads raw sales records from MySQL via JDBC, aggregates them in-memory using Java Collections, and serves the results to a responsive Chart.js dashboard — all without a single external framework.

```
MySQL ──JDBC──▶ SalesDAO ──▶ AggregationService ──▶ dashboard-data.json ──fetch()──▶ Chart.js
                 (1 query)     (HashMap/TreeMap)        (Java HttpServer)              (4 charts)
```

---

## ✨ Features

- **Single JDBC round trip** feeds all four charts — no N+1 queries, no redundant API calls
- **In-memory aggregation** via `HashMap`, `TreeMap`, and `ArrayList` — no SQL `GROUP BY` per chart
- **Zero-framework backend** — plain JDK + MySQL Connector/J only
- **Built-in HTTP server** using `com.sun.net.httpserver` — no Tomcat or Spring required
- **Live refresh** — click a button to re-run the full Java pipeline without restarting the server
- **Responsive dark UI** — CSS Grid layout, works on desktop and mobile

### Charts included

| Chart | Type | Data source |
|-------|------|-------------|
| Revenue trend | Line | `revenueByMonth` (TreeMap) |
| Revenue by category | Doughnut | `revenueByCategory` (HashMap) |
| Revenue by region | Bar | `revenueByRegion` (HashMap) |
| Top products by units | Horizontal bar | `topProducts` (sorted List) |

## 🛠️ Tech Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| Database | MySQL 8.x | Relational, widely used, JDBC-native |
| Data access | JDBC (raw) | No ORM overhead; shows low-level DB skills |
| Aggregation | Java Collections (`HashMap`, `TreeMap`, `ArrayList`) | In-memory processing, O(n) single pass |
| JSON output | Hand-written `JsonWriter` | Zero external dependencies |
| HTTP server | `com.sun.net.httpserver` | Built into the JDK, no framework needed |
| Frontend | HTML5 + CSS3 + Vanilla JS | No build tooling required |
| Charts | Chart.js 4.4 (CDN) | Responsive, well-documented, lightweight |

## ⚙️ Prerequisites

Before you begin, make sure you have:

## 🚀 Setup & Installation

### Step 1 — Clone the repository

```bash
git clone https://github.com/naveenkumargummagatta-2005/analytics-dashboard.git
cd analytics-dashboard
```

### Step 2 — Download the JDBC driver

1. Go to https://dev.mysql.com/downloads/connector/j/
2. Select **Platform Independent** → download the ZIP or TAR
3. Extract it and copy the jar into the project's `lib/` folder:

```bash
cp mysql-connector-j-8.x.x.jar lib/
```

### Step 3 — Set up the database

```bash
mysql -u root -p < sql/schema.sql
```

This creates `analytics_db`, the `sales` table, and inserts 40 sample rows
across 6 months, 3 categories, and 4 regions. Verify with:

```sql
USE analytics_db;
SELECT COUNT(*) FROM sales;   -- expected: 40
```

### Step 4 — Configure the connection

Edit `src/db.properties`:

```properties
db.url=jdbc:mysql://localhost:3306/analytics_db?useSSL=false&serverTimezone=UTC
db.user=root
db.password=YOUR_MYSQL_PASSWORD_HERE
```

### Step 5 — Compile

```bash
# macOS / Linux
mkdir -p bin
javac -cp "lib/*" -d bin $(find src -name "*.java")
cp src/db.properties bin/
```

```bat
:: Windows (Command Prompt)
mkdir bin
for /r src %f in (*.java) do @echo %f >> sources.txt
javac -cp "lib/*" -d bin @sources.txt
copy src\db.properties bin\
```

### Step 6 — Run the data pipeline

```bash
# macOS / Linux
java -cp "bin:lib/*" com.dashboard.DataPipeline
```

```bat
:: Windows
java -cp "bin;lib/*" com.dashboard.DataPipeline
```

Expected output:
```
Pipeline finished. JSON written to output/dashboard-data.json
```

### Step 7 — Start the web server

```bash
# macOS / Linux
java -cp "bin:lib/*" com.dashboard.server.DashboardServer
```

```bat
:: Windows
java -cp "bin;lib/*" com.dashboard.server.DashboardServer
```

Expected output:
```
Dashboard server running at http://localhost:8080
```

Open **http://localhost:8080** in your browser. 🎉

> Use the **"Re-run pipeline"** button in the top bar to re-query the database
> and refresh all charts live — no server restart needed.

## 🔄 How the Pipeline Works

```
┌─────────────┐      one SELECT       ┌──────────────┐
│   MySQL DB  │ ─────────────────────▶│  SalesDAO    │
│  (sales)    │       via JDBC        │  findAll()   │
└─────────────┘                       └──────┬───────┘
                                             │ List<SaleRecord>
                                             ▼
                                    ┌─────────────────────┐
                                    │  AggregationService │
                                    │                     │
                                    │  HashMap  → by cat  │
                                    │  HashMap  → by reg  │
                                    │  TreeMap  → by month│
                                    │  List     → top prod│
                                    └──────────┬──────────┘
                                               │ aggregated maps
                                               ▼
                                    ┌──────────────────────┐
                                    │   DataPipeline       │
                                    │   JsonWriter         │
                                    │   → dashboard-data   │
                                    │     .json            │
                                    └──────────┬───────────┘
                                               │ HTTP GET /api/data
                                               ▼
                                    ┌──────────────────────┐
                                    │   Chart.js frontend  │
                                    │   4 charts + KPIs    │
                                    └──────────────────────┘
```

**Key design decision:** All four charts derive from one JDBC query.
`AggregationService` makes a single O(n) pass through `List<SaleRecord>`
and populates all grouping maps simultaneously — the browser never makes
more than one `/api/data` fetch.

## 🐛 Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `Cannot load driver class: com.mysql.cj.jdbc.Driver` | JAR missing from classpath | Confirm jar is in `lib/` and you used `-cp "lib/*"` |
| `Access denied for user 'root'@'localhost'` | Wrong password in db.properties | Update `db.password` |
| `db.properties not found on classpath` | Properties file not copied to `bin/` | Run `cp src/db.properties bin/` |
| Browser shows "could not load data" | JSON not generated yet | Run `DataPipeline` first (Step 6) |
| `Address already in use: 8080` | Port conflict | Change `PORT` in `DashboardServer.java` and recompile |
