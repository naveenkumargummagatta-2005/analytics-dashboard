# Sales Analytics Dashboard тАФ Java + JDBC + Chart.js

A full-stack analytics pipeline:

```
MySQL  --JDBC-->  Java backend (Collections aggregation)  --JSON file-->  Java HTTP server  --fetch()-->  Chart.js frontend
```

No frameworks (no Spring, no Maven required) тАФ just the JDK, the MySQL JDBC
driver jar, and a browser. This README walks through everything from a
completely empty machine.

## 2. Prerequisites

You need three things installed:

1. **JDK 11 or later** тАФ check with `java -version` and `javac -version`.
   If `javac` is missing, you only have a JRE; install a full JDK
   (Temurin/OpenJDK from https://adoptium.net).
2. **MySQL Server 8.x** running locally (or any reachable MySQL instance).
3. **MySQL Connector/J** (the JDBC driver jar) тАФ download the "Platform
   Independent" ZIP/TAR from https://dev.mysql.com/downloads/connector/j/,
   extract it, and copy the file named like `mysql-connector-j-8.x.x.jar`
   into the project's `lib/` folder.

---

## 3. Set up the database

Open a MySQL client (e.g. `mysql -u root -p`) and run the schema script:

```bash
mysql -u root -p < sql/schema.sql
```

This creates the `analytics_db` database, a `sales` table, and inserts
~40 sample rows spanning 6 months, 3 categories (Electronics, Furniture,
Stationery), and 4 regions (North, South, East, West).

Verify it worked:

```sql
USE analytics_db;
SELECT COUNT(*) FROM sales;   -- should return 40
```

---

## 4. Configure the connection

Edit `src/db.properties` and put in your real MySQL password:

```properties
db.url=jdbc:mysql://localhost:3306/analytics_db?useSSL=false&serverTimezone=UTC
db.user=root
db.password=YOUR_MYSQL_PASSWORD_HERE
```

If MySQL runs on a different host/port, or you used a different schema
name, adjust `db.url` accordingly.

---

## 5. Compile the Java backend

From the project root (`analytics-dashboard/`), run:

```bash
mkdir -p bin
javac -cp "lib/*" -d bin $(find src -name "*.java")
cp src/db.properties bin/
```

What this does:
- `-cp "lib/*"` puts the JDBC driver jar on the classpath so `javac` can
  resolve `java.sql.*` driver classes at runtime (the `java.sql` package
  itself is built into the JDK; the jar provides the actual MySQL driver
  implementation that `DriverManager` loads).
- `-d bin` places compiled `.class` files in `bin/`, mirroring the package
  structure (`bin/com/dashboard/...`).
- Copying `db.properties` into `bin/` puts it on the classpath at runtime,
  which is where `DatabaseConnection` looks for it.

On Windows (Command Prompt), the wildcard expansion differs slightly:

```bat
mkdir bin
for /r src %f in (*.java) do @echo %f >> sources.txt
javac -cp "lib/*" -d bin @sources.txt
copy src\db.properties bin\
```

## 6. Run the data pipeline (Java + JDBC + Collections тЖТ JSON)

This step connects to MySQL, pulls every row, aggregates it with
`HashMap`/`TreeMap`, and writes `output/dashboard-data.json`.

```bash
java -cp "bin:lib/*" com.dashboard.DataPipeline
```

(On Windows, classpath separators are `;` not `:`):

```bat
java -cp "bin;lib/*" com.dashboard.DataPipeline
```

Expected output:

```
Pipeline finished. JSON written to output/dashboard-data.json
```

Open `output/dashboard-data.json` to see the aggregated structure:
total revenue, revenue by category/region/month, and top products by
units sold тАФ all computed in-memory from a single JDBC query.

---

## 7. Start the web server

This serves the JSON via `/api/data` and the frontend files from `web/`,
using Java's built-in `HttpServer` (no Tomcat/Spring needed).

```bash
java -cp "bin:lib/*" com.dashboard.server.DashboardServer
```

You should see:

```
Dashboard server running at http://localhost:8080
```

Open **http://localhost:8080** in your browser. The dashboard loads,
fetches the JSON, and renders four Chart.js charts plus KPI cards.

Click **"Re-run pipeline"** in the top bar at any time тАФ it calls
`/api/refresh`, which re-runs the exact same JDBC + Collections pipeline
server-side and reloads the charts with fresh data, without restarting
the server.

## 8. How the pieces fit together (for your resume talking points)

- **JDBC**: `SalesDAO.findAll()` issues exactly one `SELECT` for the whole
  dataset тАФ no per-row queries, no N+1 problem. `DatabaseConnection`
  centralizes credentials so connection logic lives in one place.
- **Collections**: `AggregationService` uses `HashMap`/`LinkedHashMap` for
  unordered grouping (by category, region, product) and `TreeMap` for the
  monthly trend, which needs chronological order тАФ letting the map itself
  guarantee sort order instead of sorting afterward.
- **Minimised redundant calls**: all four chart datasets (trend, category,
  region, top products) are derived from a *single* JDBC round trip and a
  single JSON file fetch on the frontend тАФ the browser doesn't make four
  separate API calls for four charts.
- **JSON generation**: `JsonWriter` is a small hand-written serializer
  (no external library), keeping the project dependency-free aside from
  the JDBC driver itself.
- **Chart.js frontend**: `dashboard.js` fetches once and renders a line
  chart (trend), doughnut (category split), bar chart (region), and a
  horizontal bar chart (top products) тАФ all responsive via Chart.js's
  built-in resize handling and the page's CSS grid.

## 9. Common issues

**`Cannot load driver class: com.mysql.cj.jdbc.Driver`**
The jar isn't on the classpath. Confirm `lib/mysql-connector-j-*.jar`
exists and that you included `-cp "lib/*"` in both the `javac` and `java`
commands. Note the unquoted `*` won't expand the way you expect on every
shell тАФ always quote it as `"lib/*"`.

**`Access denied for user 'root'@'localhost'`**
Your password in `db.properties` doesn't match MySQL. Reset it or update
the file.

**`db.properties not found on classpath`**
You forgot to copy it into `bin/` after compiling, or you're running
`java` from a different working directory than expected. Re-run the
`cp src/db.properties bin/` step.

**Browser shows "could not load data"**
The JSON file doesn't exist yet тАФ run `DataPipeline` (step 6) before
starting the server, or click "Re-run pipeline" once the server is up
(it will report the underlying SQL error if the DB connection fails).

**Port 8080 already in use**
Change `PORT` at the top of `DashboardServer.java`, recompile, and
restart.
